Configuration with Non-Volatile Random-Access Memory (NVRAM)

Overview

Abbreviation of Non-Volatile Random Access Memory, a type of memory that retains its contents when power is turned off. One type of NVRAM is SRAM that is made non-volatile by connecting it to a constant power source such as a battery. Another type of NVRAM uses EEPROMchips to save its contents when power is turned off. In this case, NVRAM is composed of a combination of SRAM and EEPROM chips.
DA16200 has an NVRAM area on the flash memory to store system data and user data. NVRAM has various system configuration parameters to control the Wi-Fi function.

DA16200 Configuration

User can use APIs to set/get NVRAM parameters.

  • Set string value: int da16x_set_config_str(int name, char *value)
  • Set number value: int da16x_set_config_int(int name, int value)
  • Get string value: int da16x_get_config_str(int name, char *value)
  • Get number value: int da16x_get_config_int(int name, int *value)
    (Return: If succeeded return 0, if failed return an error code)

    [name] list configurable are written in common_config.h. (DA16X_CONF_STR / DA16X_CONF_INT)

NVRAM parameter addition for User

Users can make APIs to set/get parameter to/from NVRAM.

  1. Add a parameter name to DA16X_USER_CONF_STR or DA16X_USER_CONF_INT (enum) in the user_nvram_cmd_table.h
  2. Add a struct member to user_config_str_with_nvram_name[] or user_config_int_with_nvram_name[] in the user_nvram_cmd_table.c
    String value: { [parameter name], [NVRAM name], [Max length of the value] }
    Number value: { [parameter name], [NVRAM name], [Min value], [Max value], [Default value] }
  3. User can set/get DA16200 parameters with the configuration APIs.
