805 //@brief The function calculates user's confirmation MacTags for FFC DH Schemes according to NIST SP 56A rev.2 standard.
806 //
807 //\note Before calling of this function the user should obtain assurance of used FFC Domain and public, private keys,
808 //involved in the key agreement, using one of the methods, described in sec. 5.6.2 of above named standard.
809 //<ul><li> - depending on DH Scheme, calculates confirmation HMAC MacTag, which is intended to be provided to the partner
810 //(sec. 5.2, 5.9, 6); in this case the secret keying material is parsed to MacKey of size, equaled to HMAC key size. </li>
811 //<li> - in our implementation HMAC key size defined equal to FFC sub-group order (meets to sec.5.9.3). </li>
812 //<li> - if in the chosen DH Scheme the user is not Confirmation provider, then both the pointer and the size of
813 //appropriate MacTag should be set to NULL. </li>
814 //<li> - for detailed description of Confirmation "MacData" see CCFfcDhConfirmMacData_t structure definition. </li></ul>
815 //
816 //@return CC_OK on success.
817 //@return A non-zero value on failure as defined in cc_dh_error.h, cc_kdf_error.h or cc_hash_error.h.
818 //*/
819 //CIMPORT_C CCError_t CC_FfcDhCalcConfirmMacTags(
820 // CCFfcDhUserContext_t *pDhUserCtx, /*!< [in] pointer to the user's DH context structure, containing all data, defining
821 // DH Key Agreement Scheme. The context shall be initialized for user's roles
822 // (U or V; Provider or Receiver) using CC_FfcDhSetCtx function. */
823 // uint8_t *pUserMacTag, /*!< [out] optional, pointer to the user (provider) confirmation MacTag depending
